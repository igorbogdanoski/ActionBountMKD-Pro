# Avantura MKD Pro — User Guide

**Version:** 0.2 working draft
**Audience:** Teachers, school coordinators, students and platform administrators

## 1. What is Avantura MKD Pro?

Avantura MKD Pro is a bilingual educational platform for creating and playing location-based, classroom and hybrid learning adventures. Teachers combine quizzes, information points, missions, surveys, QR tasks, GPS challenges, branching activities and team competitions in one guided experience.

Students can participate through a mobile browser or the mobile application. Supported experiences include offline play with later synchronisation, live sessions, scoring, feedback, certificates and teacher analytics.

## 2. Before you begin

You need:

- a modern web browser;
- a teacher account for authoring and analytics;
- a mobile device with camera and location permissions for QR/GPS activities;
- internet access when downloading an adventure or synchronising offline results.

Never enter real student medical, identity-document or other sensitive personal data into an adventure.

## 3. Teacher quick start

### 3.1 Create or sign in to an account

1. Open the Avantura MKD Pro website.
2. Choose Google sign-in or email/password registration.
3. Review the privacy notice and account settings.
4. Select the preferred language and display theme.

### 3.2 Create an adventure

1. Open **Dashboard**.
2. Select **Create Adventure**.
3. Enter the title, description, learning goals and visibility.
4. Add stages and arrange them in the required order.
5. Save and preview the adventure before publishing or sharing it.

### 3.3 Choose stage types

| Stage | Best used for |
|---|---|
| Information | Instructions, context and learning material |
| Quiz | Multiple-choice, matching or ordering checks |
| Mission | Photo, audio, video or written evidence |
| Survey | Reflection and learner feedback |
| QR task | Checkpoints, exhibits and station work |
| Find Spot | GPS-based field activities |
| Switch | Conditional routes and differentiated learning |
| Tournament | Team challenges and competition |

Use short instructions, provide safe routes and always include a non-device fallback for essential learning outcomes.

### 3.4 Map stages to curriculum objectives

In the adventure's Pedagogy settings, a teacher can define stable learning objectives (for example, "solves a linear equation") and map any stage to one. A live coverage summary then shows how many stages and points are mapped to each objective, and how many stages still have no objective. Renaming an objective never breaks its mappings — they are tracked by a stable ID, not by label.

### 3.5 Configure access and sharing

Choose the appropriate visibility and play mode:

- private/secret access for controlled classroom use;
- public discovery only for content cleared for publication;
- individual play for self-paced learning;
- live session mode for teacher-led activities.

Before sharing, verify every link, media item, QR code, GPS location and answer key.

### 3.6 Run a live session

1. Open the adventure and start a host session.
2. Share the join code with students.
3. Confirm that all expected participants have joined.
4. Start the session and monitor progress.
5. Use announcements, time controls and SOS/location information responsibly.
6. Finish the session only after confirming that results have synchronised.

## 4. Student guide

### 4.1 Join an adventure

1. Open the shared link or **Join** page.
2. Enter the session code and the display name requested by the teacher.
3. Allow camera or location access only when the current task requires it.
4. Read the safety instructions before starting.

### 4.2 Play and submit evidence

- Follow the stage order shown on screen.
- Use the camera, QR scanner or GPS only in a safe location.
- Wait for a success confirmation after submitting.
- If offline mode is enabled, keep the app data until synchronisation is confirmed.
- Contact the teacher through the agreed channel if a device permission fails.

## 5. Review results

Teachers can use the Results area to:

- view rankings and completion;
- export authorised result data;
- inspect funnel drop-off by stage;
- identify weak quiz questions across adventures;
- review rubric-based mission evidence;
- provide grades and feedback.

Analytics indicate patterns; they should support, not replace, teacher judgement.

> **Illustration UG-05:** Results tabs, funnel summary and weak-spots view.

## 6. Groups and reusable templates

Use **Groups** to organise classes, assignments and learner rosters. For any assigned adventure that has curriculum objectives mapped, a teacher can export a per-student objective-mastery CSV report directly from the group's page — one row per student, one column per objective, showing the share of that objective's stages each student actually reached. Use **Templates** to start from a reviewed pedagogical structure. Always adapt a template to the learners, curriculum, location and accessibility requirements.

## 7. Offline use

1. Download the adventure while online.
2. Confirm that the offline indicator reports success.
3. Keep the browser/app storage intact during the activity.
4. Reconnect after the session.
5. Wait for queued results to synchronise before clearing application data.

Offline access does not guarantee that every external media link will work without connectivity.

## 8. Accessibility and safeguarding

- Provide alternatives for GPS, camera, audio and timed tasks.
- Avoid routes with unsafe traffic, inaccessible terrain or restricted areas.
- Use high-contrast text and concise instructions.
- Do not publish student names, images or precise locations without the required consent.
- Use school-approved devices and safeguarding procedures.

## 9. Troubleshooting

| Problem | Recommended action |
|---|---|
| Join code is rejected | Check spelling, session status and internet connection |
| Camera/QR does not open | Allow camera permission and close other camera apps |
| GPS position is inaccurate | Move outdoors, enable precise location and retry |
| Result is waiting to sync | Reconnect and keep the app open until confirmation |
| Teacher cannot export | Confirm plan permissions and that results exist |
| AI generation fails | Retry with a shorter prompt; do not include personal data |

## 10. Administrator checklist

- Review account roles and administrator claims.
- Publish only approved templates.
- Review payment requests through the documented moderation workflow.
- Monitor error reports and suspicious activity.
- Maintain privacy, retention, backup and incident procedures.
- Test restore and business-continuity processes periodically.
